# 스타일 DNA

## 한 문장

순백, 미니멀, 손그림, 여백, 절제, 괴상함, 제품 스케치 느낌, 한국어 손글씨감, 구조는 명확하지만 설명서 같지 않은 그림.

AI, 제품, 디자인, 개발 도구를 오래 다룬 사람이 흰 종이에 즉흥적으로 그린 설명 스케치처럼 보이게 한다.

## 필수

- 16:9 가로형 본문 삽화.
- 순백색 배경: 미색, 웜그레이, 종이 질감, 그라디언트, 그림자, 노이즈, 빈티지 종이 느낌 금지.
- 검은색 손그림 선화를 중심으로 한다: 얇은 선, 살짝 흔들리는 펜선, 기계적이지 않고 벡터 같지 않으며 두꺼운 외곽선이 아니다.
- 넉넉한 여백: 주 피사체는 화면의 약 40%-60%, 최소 35%는 빈 공간, 가능하면 조용한 여백 덩어리 하나를 둔다.
- 소량의 한국어 손글씨 주석: 최대 5-8곳, 각 주석은 되도록 2-8글자.
- 한 그림은 하나의 핵심 동작, 구조, 상태, 은유만 말한다.
- 구조는 자연스럽게 표현하고, 그림 위에 구조 유형 이름을 쓰지 않는다.

## 색

- 검정: 주요 선화, 캐릭터, 박스선, 구조, 주요 문자, 주요 사물.
- 빨강: 핵심 주석, 문제, 감정 지점, 중요한 경고, 결과.
- 주황: 주 흐름, 경로, 화살표, 자동화 흐름, A에서 B로 이동하는 관계.
- 파랑: 보조 설명, 머릿속 상태, 시스템 상태, 두 번째 층의 설명, AI/assistant/자동화 힌트.

파랑은 모든 그림에 꼭 필요하지 않다. 색은 절제하고, 많기보다는 적게 쓴다.

## 절대 금지

- 상업 일러스트 금지.
- PPT 인포그래픽 금지.
- 공식 흐름도 금지.
- 강의 자료 느낌 금지.
- 귀여운 만화 포스터 금지.
- 어린이 삽화 금지.
- 복잡한 아키텍처 다이어그램 금지.
- 정교한 플랫 일러스트 금지.
- 테크 UI 느낌 금지.
- 실제 앱 스크린샷 금지.
- 복잡한 배경, 그라디언트, 그림자, 질감 금지.
- 모든 노드를 설명하려고 하지 않는다.
- 좌상단에 “Workflow / 흐름도 / 시스템 아키텍처 / 흔한 함정 / 로드맵” 같은 유형 제목을 쓰지 않는다.
- 중국어 라벨과 영어 설명 라벨을 쓰지 않는다. 이미지 안 텍스트는 한국어만 사용한다.

## 미감 방향

괴상하고, 창의적이고, 재미있고, 간결하고 산뜻하며, 상상력이 살아 있어야 한다.

귀엽거나, 유치하거나, 너무 복잡하거나, 딱딱하면 안 된다.
